# Web

SvelteKit frontend for the [template-onchain-app](../README.md) monorepo. The app talks to the contracts package via a generated deployments file and is built as a static site for IPFS-style hosting.

## Stack

- **[SvelteKit 2](https://kit.svelte.dev/) + [Svelte 5](https://svelte.dev/)** with `@sveltejs/adapter-static` for fully static, relative-path output suitable for IPFS gateways.
- **[Tailwind CSS 4](https://tailwindcss.com/)** via the Vite plugin, plus `@tailwindcss/forms`, `@tailwindcss/typography`, and `tw-animate-css`.
- **[shadcn-svelte](https://shadcn-svelte.com/)** components (configured via `components.json`); primitives from `bits-ui`, `vaul-svelte`, `mode-watcher`, and `svelte-sonner`.
- **PWA**: icons generated by [`pwag`](https://github.com/wighawag/pwag) from `static/icon.svg` + `src/web-config.json`; service worker in `src/service-worker/`.
- **Onchain integration** via the [@etherkit](https://github.com/etherkit) and [@etherplay](https://github.com/etherplay) packages:
  - `@etherplay/connect` — wallet connection / EIP-1193 multiplexing.
  - `@etherkit/burner-wallet` — local burner wallet for development.
  - `@etherkit/tx-observer` and `@etherkit/viem-tx-tracker` — transaction lifecycle tracking.
  - `viem` for RPC calls; `eip-1193` for transport types.
- **Reactive state** via `observator` / `@observator/svelte`, with `synqable` and `radiate` helpers.
- **Testing**: [Vitest 4](https://vitest.dev/) with the Playwright browser provider for unit tests, and [Playwright](https://playwright.dev/) for end-to-end tests.

## Generated Deployments

`src/lib/deployments.ts` is generated by `rocketh-export` from the `contracts/` package. **Do not edit it by hand** — regenerate it instead:

```bash
# from the monorepo root
pnpm contracts:export <network> --ts ../web/src/lib/deployments.ts
```

It exposes addresses and ABIs for use in the app:

```typescript
import deployments from '$lib/deployments';

const {address, abi} = deployments.contracts.GreetingsRegistry;
```

When `pnpm start` (or `pnpm attach <network>` / `pnpm remote-chain <network>`) is running, the deployments file is regenerated automatically as contracts change, and Vite HMR reloads the app.

## Configuration

- **App identity** — edit [`src/web-config.json`](./src/web-config.json) to set the name, title, description, canonical URL, and theme colour. PWA icons are regenerated from this on `prepare`.
- **shadcn-svelte** — [`components.json`](./components.json) controls component placement (under `$lib/shadcn`).
- **SvelteKit** — [`svelte.config.js`](./svelte.config.js) configures `adapter-static` with relative paths so the build can be served from any subpath (IPFS, GitHub Pages, etc.).

## Scripts

| Script                    | Description                                                                                       |
| ------------------------- | ------------------------------------------------------------------------------------------------- |
| `dev`                     | Run Vite in dev mode. Proxied through `ldenv` so `pnpm dev <mode>` selects an env.                |
| `build [mode]`            | Build the static site to `build/`.                                                                |
| `preview`                 | Serve the dev preview (`vite preview`).                                                           |
| `serve`                   | Serve `build/` through `ipfs-emulator` on port 8080 to mimic IPFS gateway behaviour.              |
| `check` / `check:watch`   | Run `svelte-check` over the project.                                                              |
| `format` / `format:check` | Prettier with `prettier-plugin-svelte` and `prettier-plugin-tailwindcss`.                         |
| `test:unit`               | Vitest in browser mode (Playwright provider).                                                     |
| `test:e2e`                | Playwright e2e suite (see [`e2e/`](./e2e/) and [`playwright.config.ts`](./playwright.config.ts)). |
| `test`                    | Run unit tests once, then e2e.                                                                    |
| `generate-pwa-icons`      | Run `pwag` against `static/icon.svg` + `src/web-config.json`. Invoked on `prepare`.               |

The `pretest:e2e` hook frees port 4173 before Playwright starts its own preview server.

## End-to-end tests

Playwright drives the built preview server, configured via [`playwright.config.ts`](./playwright.config.ts). For a full pipeline (start a node, deploy contracts, build the web app, run e2e), see [`scripts/run-e2e-tests.sh`](../scripts/README.md) at the repo root.

## Deployment

The static `build/` output is suitable for IPFS pinning, decentralised storage, or any static host. Because the SvelteKit static adapter is configured with relative paths, no base path needs to be set when the site is served from a CID subpath.
